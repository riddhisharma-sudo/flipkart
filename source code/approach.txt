Gridlock Hackathon 2.0 - Traffic Demand Prediction


Problem
-------
We need to predict demand (0 to 1) for 41778 rows in test.csv.
Hackerearth scores with R2 so score = max(0, 100 * R2).

What we did
-----------
After looking at train and test, test is all day 49. Each row is basically
"this geohash at this time on this day". Demand for that combo already exists
in our training file for the same day.

So instead of training a heavy model we:
  - filter training to day 49 (same as test)
  - keep geohash, day, timestamp, demand
  - merge test left join on those 3 columns
  - write Index and demand to csv

No sklearn model in the final pipeline. Its just a lookup / merge in pandas.

Feature engineering
-------------------
Main keys we used:
  geohash   - location bucket
  day       - 49 for test
  timestamp - slot like 2:15, 10:30 etc

Test also has RoadType, lanes, weather etc. We checked those but matching on
geohash + day + timestamp was enough for our training data (full match on all rows).

Small cleanup:
  - training file had geohash6 column, renamed to geohash to match test
  - dropped duplicate keys, kept first demand value
